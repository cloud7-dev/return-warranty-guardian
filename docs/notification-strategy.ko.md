# 알림 전략

Return & Warranty Guardian은 현재 로컬 웹앱입니다. 구매내역을 서버에 업로드하지 않는다는 원칙을 지키면, 모바일과 PC에서 "앱이 닫혀 있어도 항상 울리는" 자체 푸시 알림은 브라우저 제약 때문에 제한됩니다.

## 현재 가능한 방식

1. `.ics` 캘린더 내보내기
   - 이미 구현되어 있습니다.
   - 구매별 사전 알림 일수를 저장하고, 내보낸 `.ics` 이벤트에 반복 `VALARM` 알림을 포함합니다.
   - 앱 안의 `캘린더 알림` 가이드에서 모바일/PC 가져오기 경로를 바로 확인할 수 있습니다.
   - 모바일에서는 iOS 캘린더, Google Calendar, Samsung Calendar 등에 추가하면 OS 캘린더 알림을 받을 수 있습니다.
   - PC에서는 macOS Calendar, Outlook, Google Calendar, Windows Calendar에서 알림을 받을 수 있습니다.
   - 서버 없이 가장 안정적인 알림 경로입니다.

2. 앱을 열었을 때 마감 큐 확인
   - 현재 대시보드는 곧 마감, 만료, 증빙 부족 항목을 즉시 보여줍니다.
   - 브라우저 탭이나 PWA가 열려 있는 동안에는 `로컬 알림` 버튼으로 Web Notifications API 기반 보조 알림을 켤 수 있습니다.
   - 앱 안의 알림 큐에서 개별 마감을 1일 또는 7일 스누즈할 수 있고, 스누즈를 전체 해제할 수 있습니다.

## 모바일 알림

권장 MVP 흐름은 PWA 설치 + 캘린더 알림입니다.

1. 사용자가 구매내역을 등록합니다.
2. 앱이 반품기한, 환불 가능 기간, 보증기간을 계산합니다.
3. 사용자가 구매별 사전 알림 일수를 확인합니다.
4. 앱 안의 `캘린더 알림` 가이드에서 사용할 캘린더 경로를 확인합니다.
5. 사용자가 `ICS`를 내보냅니다.
6. 휴대폰 캘린더에 가져옵니다.
7. 캘린더 앱이 `VALARM` 기준으로 마감 전 알림을 보냅니다.

iOS와 Android의 Web Push는 가능하지만 안정적인 백그라운드 푸시에는 보통 푸시 서버, VAPID 키, 구독 엔드포인트 저장이 필요합니다. 이 방식은 "서버 업로드 없음" 메시지와 충돌할 수 있으므로 기본 MVP에는 넣지 않는 편이 맞습니다.

## PC 알림

권장 MVP 흐름은 데스크톱 캘린더 또는 브라우저 내 알림입니다.

1. 캘린더 앱을 쓰는 사용자는 `.ics` 파일을 가져와 OS 알림을 받습니다.
2. 앱을 자주 여는 사용자는 대시보드의 곧 마감/만료 상태를 확인합니다.
3. 앱이 열려 있을 때만 동작하는 브라우저 로컬 알림을 켤 수 있습니다.
4. 방금 처리하기 어려운 항목은 앱 안에서 3시간, 내일까지, 7일 스누즈합니다.

## 향후 옵션

### A. 서버 없는 기본 옵션

- `.ics` 반복 내보내기 개선
- 마감 N일 전 알림값 선택 및 `VALARM` 반영
- 캘린더 이벤트 제목/설명 커스터마이즈
- 앱이 열려 있을 때만 동작하는 로컬 브라우저 알림
- 앱 안의 3시간/내일까지/7일 스누즈와 스누즈 해제

이 옵션은 privacy-friendly 메시지와 가장 잘 맞습니다.

### B. 선택형 푸시 연동

- ntfy, Gotify, Apprise 같은 사용자가 직접 관리하는 알림 채널과 연동
- 사용자가 provider, endpoint, topic을 직접 입력
- 구매내역 원문 대신 마감일, 별칭, 익명화된 제목만 전송
- 현재 구현은 provider/endpoint/topic 설정을 로컬에만 저장하고, 실제 전송이 아니라 ntfy/Gotify/Apprise용 JSON/curl 초안과 dry-run report를 로컬에서 내보냅니다. 토큰은 저장하지 않습니다.

이 옵션은 고급 사용자에게 좋지만 초기 MVP에는 과합니다.

### C. 호스팅 푸시 서비스

- 자체 푸시 서버 운영
- Web Push 구독 저장
- 모바일/PC 모두 백그라운드 알림 지원

이 옵션은 사용성은 좋지만 서버 운영, 개인정보 신뢰, 보안 책임이 생깁니다. 현재 제품 방향인 "서버 업로드 없음"과는 거리가 있습니다.

## 추천 결론

현재 기본 알림 경로는 `.ics` 캘린더 알림입니다. 앱 안에서는 구매별 사전 알림 일수를 설정하고, `.ics`에는 반복 `VALARM`을 포함합니다. 브라우저 알림과 1일/7일 스누즈는 앱이 열려 있을 때의 보조 알림으로만 사용합니다. self-hosted 알림은 현재 payload 초안 내보내기까지만 제공하고, 실제 전송 설정은 명시적 opt-in 기능으로 분리합니다.
